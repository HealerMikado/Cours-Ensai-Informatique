Tour de table

Compétences visées

  • Développer une application complexe en python (base de données, service web, données non tabulaire)

  • Valoriser son travail à l'écrit et à l'oral

Cadre général

  • Groupe de 4-5 élèves (quelques à 6)
  • 4 groupes par tuteur
  • Sujet de votre choix
  • Tout le premier semestre de 1A (100h/élèves)
  • 2 phases :
    1. Analyse/conception générale (3 séances)
    2. Implémentation (4 séances)

Les sujets

  • Communication avec une base de données
  • Communication avec des données sur du web (de préférence une API)
  • Manipulation de données sous un format autre qu'une table (JSON, XML, ... ie. no-SQL)
  • Modélisation du projet à l'aide de diagrammes UML
Régulièrement les projets ne collent pas avec toutes ces attentes à la fois, on rajoute alors une fonctionnalité / un cas d'usage qui comble le point manquant. Avec un peu d'imagination on trouve toujours une solution.
